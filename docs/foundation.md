# Goal

Create a strong foundation before building any UI.

The objective was to choose a modern technology stack, organize the project structure, and establish coding standards so future development would be easier and more maintainable.

---

# What We Built

## Project Setup

- Created a Next.js project using the App Router.
- Configured TypeScript.
- Configured Tailwind CSS.
- Installed shadcn/ui.
- Added next-themes for dark/light mode support.
- Added Lucide React and Tabler Icons.
- Configured npm as the package manager.

---

## Folder Structure

Created the base project structure.

```text
app/
components/
config/
content/
hooks/
lib/
public/
styles/
types/
```

Each folder has a single responsibility to keep the project organized.

# Decisions Made

### Why Next.js?

- Modern React framework.
- App Router support.
- Better SEO.
- Better performance.
- Industry standard for production React applications.

---

### Why TypeScript?

- Better developer experience.
- Type safety.
- Easier refactoring.
- Fewer runtime errors.

---

### Why Tailwind CSS?

- Faster UI development.
- Consistent styling.
- Utility-first workflow.
- No separate CSS files for most components.

---

### Why shadcn/ui?

- Accessible components.
- Easy customization.
- Works naturally with Tailwind CSS.
- No unnecessary styling.

---

# Project Rules

- Build first.
- Refactor second.
- Polish third.

- Prefer reusable components.

- Keep UI separate from content.

- Keep components small.

- Run `npm run build` before committing.
